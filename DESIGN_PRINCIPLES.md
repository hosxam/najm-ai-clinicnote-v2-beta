# Design Principles

## Doctor-first

- Replace confusing technical modes with a straightforward documentation flow.
- Keep the main prompt centered on the doctor’s task, not the internal schema.

## Low-friction workflow choice

- One main workflow search box on the home screen
- Common workflow suggestions for faster starts
- Excluded workflows hidden during limited testing

## Safety-forward drafting

- No diagnosis suggestions
- No treatment suggestions
- No medication doses
- No invented findings
- No clinician plan generated unless entered or explicitly selected by the clinician

## Progressive depth

- Quick Note for fast OPD documentation
- Detailed Encounter for richer structured drafting
- Medical Report / Letter for simple documentation requests

## Clear language

- Use labels like Symptoms / History, Examination, Investigations, Assessment, Plan, and Output
- Do not expose internal schema terms to normal users
